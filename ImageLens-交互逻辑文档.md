Interaction Logic Document

版本：v2.0

**文档概述**

本文档详细描述 ImageLens 图片智能查看应用的所有界面交互逻辑，包括组件状态定义、用户操作场景、事件触发条件、反馈表现和边界情况处理。

应用采用 Electron 架构，分为主进程（数据处理）和渲染进程（UI 展示）。本文档仅覆盖渲染进程的前端交互逻辑。

**功能模块总览**

  -------------- -------------- --------------------------------------------------
  **模块**       **对应页面**   **核心交互**
  图片浏览       browse         导入、搜索、筛选、布局切换、单击详情、双击预览
  图片详情面板   ---            Tab 切换、AI 维度展示、标签管理、OCR 文字展示
  全屏预览       ---            键盘左右箭头、鼠标点击箭头、滚轮上下、ESC 关闭
  导入图片       ---            文件夹/单文件选择、MD5 去重、异步处理
  进度面板       ---            状态栏总进度、展开面板、四线状态、重试/暂停/取消
  人物管理       people         命名、合并、拆分、按人物筛选
  插件管理       plugins        安装、启用/禁用、卸载
  设置           settings       AI 模式、OCR 方案、压缩参数、缓存管理
  -------------- -------------- --------------------------------------------------

**1. 图片浏览**

**1.1 侧边栏导航**

**组件状态定义**

  ---------- ------------------------------------------------
  **状态**   **表现**
  Default    当前页面对应的 nav-item 无高亮
  Active     当前页面对应的 nav-item 显示青色背景和白色文字
  ---------- ------------------------------------------------

**交互逻辑**

**场景A：点击导航项**

-   触发：nav-item 的 onclick 事件

-   动作：调用 switchPage(page)，移除所有 .page 和 .nav-item 的 .active 类，给目标页面和导航项添加 .active

-   反馈：主内容区切换到对应页面，侧边栏导航项高亮移动

**场景B：点击智能分类项（风景/人像/美食等）**

-   触发：onclick 调用 filterByCategory(cat)

-   动作：先调用 switchPage(\'browse\') 确保回到浏览页，然后按分类过滤图片网格

-   反馈：图片网格只显示该分类的图片

**1.2 图片卡片**

**组件状态定义**

  ---------- -------------------------------------------------------
  **状态**   **表现**
  Default    图片缩略图 + 文件名 + AI 维度指示点，无悬浮效果
  Hover      出现半透明遮罩层，显示文件名和前 3 个标签，卡片微上移
  ---------- -------------------------------------------------------

**交互逻辑**

**场景A：单击图片卡片**

-   触发：.image-card 的 onclick 事件

-   动作：调用 openDetail(id)，根据 id 查找图片数据，填充详情面板内容，给 \#detailPanel 添加 .open 类

-   反馈：右侧详情面板从右侧滑入，展示当前图片的 AI 识别结果

**场景B：双击图片卡片**

-   触发：.image-card 的 ondblclick 事件

-   动作：调用 openPreview(id)，设置 currentPreviewIndex 和 isPreviewOpen=true，更新预览内容，给 \#previewModal 添加 .open

-   反馈：全屏黑色遮罩弹出，展示原图大图、文件名、计数器、左右箭头

**注意：单击和双击的区分**

-   浏览器会先触发两次 onclick，再触发一次 ondblclick

-   单击打开详情面板后，双击仍会打开预览（预览覆盖在详情之上）

**1.3 筛选芯片（Chip）**

**组件状态定义**

  ---------- --------------------------
  **状态**   **表现**
  Default    透明背景，灰色文字和图标
  Active     青色背景，白色文字和图标
  ---------- --------------------------

**交互逻辑**

**场景：点击筛选芯片**

-   触发：.chip 的 onclick 调用 setFilter(this)

-   动作：移除所有 .chip 的 .active，给当前点击的 chip 添加 .active（互斥选中）

-   反馈：选中的芯片高亮，图片网格按对应维度过滤

-   可用维度：全部 / 物体 / 场景 / 活动 / 情绪 / 文字

**1.4 搜索框**

**组件状态定义**

  ---------- --------------------------------------------
  **状态**   **表现**
  Default    空输入框，占位符"搜索图片..."
  Typing     用户输入关键词，实时过滤图片网格
  Result     图片网格只显示匹配的图片，匹配文字片段高亮
  ---------- --------------------------------------------

**交互逻辑**

**场景：用户输入关键词**

-   触发：input 的 oninput 事件

-   动作：同时搜索文件名、AI 标签、分类、图片内文字（FTS5 全文索引）

-   反馈：图片网格实时更新，搜索结果中高亮匹配的文字片段

-   边界：输入为空时恢复显示全部图片

**2. 图片详情面板**

**2.1 面板开关**

**组件状态定义**

  ---------- ------------------------------------------------
  **状态**   **表现**
  Closed     面板完全隐藏在右侧屏幕外（translateX(100%)）
  Open       面板从右侧滑入，宽度约 380px，压缩图片网格区域
  ---------- ------------------------------------------------

**交互逻辑**

**场景A：打开详情面板**

-   触发：单击图片卡片，调用 openDetail(id)

-   动作：查找图片数据，填充缩略图/文件名/元信息，添加 .open 类，调用 renderDetailContent() 渲染当前 Tab 内容

**场景B：关闭详情面板**

-   触发：点击关闭按钮（× 图标），调用 closeDetail()

-   动作：移除 \#detailPanel 的 .open 类

-   反馈：面板向右滑出并隐藏

**2.2 Tab 切换**

**组件状态定义**

  ---------- -------------- ------------------------------------------------------
  **Tab**    **对应参数**   **展示内容**
  AI 识别    ai             自动分类、五维度进度条、物体列表、标签、识别文字
  元数据     exif           相机信息、拍摄参数、位置与时间
  文件信息   info           文件名、格式、尺寸、大小、MD5、导入时间、AI 识别信息
  ---------- -------------- ------------------------------------------------------

**交互逻辑**

**场景：点击 Tab 标签**

-   触发：.detail-tab 的 onclick 调用 switchDetailTab(el, tab)

-   动作：移除所有 .detail-tab 的 .active，给当前 tab 添加 .active，更新全局变量 currentDetailTab，重新调用 renderDetailContent()

-   反馈：当前 Tab 高亮，面板内容区域切换到对应内容

**2.3 AI 识别 Tab 内容**

**维度进度条**

每个 AI 识别维度（物体/场景/活动/情绪/文字）显示为一个进度条：

  ---------------- ------------------ ----------
  **置信度**       **进度条颜色**     **宽度**
  ≥ 0.8            绿色（high）       ≥ 80%
  ≥ 0.6            黄色（medium）     ≥ 60%
  \< 0.6           红色（low）        \< 60%
  未识别（null）   灰色文字"未识别"   无进度条
  ---------------- ------------------ ----------

**标签管理**

**场景：点击"+ 添加"标签**

-   触发：点击 .tag-item.user-tag

-   动作：弹出内联输入框，用户输入新标签后按 Enter 确认

-   反馈：新标签添加到标签列表末尾，同步写入数据库

**识别文字展示**

当图片包含 OCR 识别结果时，在标签列表下方显示：

-   文字内容框：展示完整识别文字，多区域用换行分隔，最大高度 200px 可滚动

-   文字区域列表：每个区域显示序号、文字内容、置信度百分比

**3. 全屏预览**

**3.1 组件状态定义**

  ----------- ----------------------------------------------------------------------
  **状态**    **表现**
  Closed      全屏遮罩隐藏，isPreviewOpen = false
  Open        全屏黑色背景，居中展示原图大图，顶部文件名、底部计数器、左右箭头按钮
  Switching   图片切换动画中，当前图片 opacity 降低到 0.3，100ms 后更换并恢复
  ----------- ----------------------------------------------------------------------

**3.2 交互逻辑（按时间顺序）**

**场景A：打开预览**

-   触发：双击图片卡片，调用 openPreview(id)

-   动作：设置 isPreviewOpen=true，记录 currentPreviewIndex，更新预览内容（图片源/文件名/计数器），添加 .open

**场景B：键盘左箭头切换上一张**

-   触发：全局 keydown 监听，判断 key === \'ArrowLeft\' 且 isPreviewOpen === true

-   动作：调用 navigatePreview(-1)，当前索引 -1（取模循环），图片添加 .switching 类，100ms 后更新并移除

-   边界：第一张图片向前切换时，自动跳转到最后一张（循环）

**场景C：键盘右箭头切换下一张**

-   触发：key === \'ArrowRight\' 且 isPreviewOpen === true

-   动作：调用 navigatePreview(1)，逻辑同场景B，方向相反

**场景D：鼠标点击左右箭头按钮**

-   触发：.preview-arrow.prev/next 的 onclick

-   动作：调用 navigatePreview(-1/1)，同时 event.stopPropagation() 阻止事件冒泡到背景遮罩

**场景E：滚轮上下滚动切换**

-   触发：\#previewModal 的 wheel 事件（passive: false）

-   动作：e.preventDefault() 阻止页面滚动，deltaY \> 0 调用 navigatePreview(1)，deltaY \< 0 调用 navigatePreview(-1)

**场景F：关闭预览**

-   触发：点击关闭按钮 / 点击背景遮罩 / 按 ESC 键

-   动作：设置 isPreviewOpen=false，重置 currentPreviewIndex=-1，移除 .open

**场景G：点击图片本身**

-   触发：.preview-img-wrapper 的 onclick

-   动作：event.stopPropagation()，不触发背景的关闭事件（图片区域点击不关闭预览）

**4. 导入图片**

**4.1 组件状态定义**

  ---------- ------------------------------------------
  **状态**   **表现**
  Closed     导入对话框隐藏
  Open       导入对话框居中弹出，包含两个导入选项卡片
  ---------- ------------------------------------------

**4.2 交互逻辑**

**场景A：点击"导入"按钮**

-   触发：.btn-primary 的 onclick 调用 openImportDialog()

-   动作：给 \#importDialog 添加 .open 类

**场景B：选择导入方式**

-   触发：.import-option 的 onclick 调用 selectImportOption(this)

-   动作：移除所有 .import-option 的 .selected，给当前选项添加 .selected（互斥选中）

-   可选项："导入文件夹"（递归扫描子文件夹）、"导入图片文件"

**场景C：点击"选择路径"按钮**

-   动作：调用 Electron 的 dialog.showOpenDialog()，文件夹模式开启 openDirectory，文件模式开启 openFile 且 filters 限制图片格式

**场景D：点击"取消"按钮**

-   触发：.btn-ghost 的 onclick 调用 closeImportDialog()

-   动作：移除 \#importDialog 的 .open 类

**场景E：导入后的自动处理**

-   扫描所有图片文件，逐张计算 MD5

-   重复图片（MD5 已存在）：仅更新 file\_path，复用已有数据，进度标记为 skipped

-   新图片：创建记录入库，异步触发四条处理线（本地处理 → 人物识别 → OCR → AI 识别）

**5. 处理进度面板**

**5.1 状态栏（底部常驻）**

**组件状态定义**

  ------------ -------------------------------------------------------------
  **状态**     **表现**
  Idle         无处理任务时，状态栏显示"已完成 N/M (100%)"，进度条绿色
  Processing   有任务处理中，显示"已完成 N/M (X%)"，进度条青色，数字可点击
  ------------ -------------------------------------------------------------

**交互逻辑**

**场景：点击状态栏进度区域**

-   触发：.statusbar-progress 的 onclick 调用 toggleProgressPanel()

-   动作：同时切换 \#progressPanel 和 \#progressBackdrop 的 .open 类

-   反馈：底部面板从下方滑出，同时出现半透明遮罩

**5.2 展开进度面板**

**面板布局**

面板从下方滑出，高度 440px，包含以下区域：

-   头部：标题"处理进度" + 批次信息 + 操作按钮组

-   总进度条：进度条 + 统计数据（完成/处理中/失败/等待中）

-   任务列表：每张图片一行，显示四条处理线的独立状态

**四线状态图标体系**

  ---------- ------------ ------------------------------
  **图标**   **含义**     **CSS 类**
  ✓          该线已完成   s-ok（绿色圆形）
  ⟳          该线处理中   s-run（青色圆形 + 脉冲动画）
  ○          该线等待中   s-wait（灰色圆形）
  ✕          该线失败     s-err（红色圆形）
  ---        该线跳过     s-skip（灰色圆形）
  ---------- ------------ ------------------------------

**任务行状态示例**

每个任务行显示格式：

photo\_001.jpg 本地 ✓ 人物 ✓(2人) OCR ✓(42字) AI ✓ 完成

IMG\_043.jpg 本地 ✓ 人物 ⟳ 识别中 OCR ○ AI ✓ 处理中

IMG\_015.jpg 本地 ✓ 人物 ✓(1人) OCR ✓(28字) AI ✕ API 超时 \[重试\] 失败

**5.3 操作按钮**

**交互逻辑**

**场景A：点击"重试失败"按钮**

-   动作：将所有失败的图片重新加入对应处理队列

-   反馈：失败图片的状态重置为 pending，重新开始处理

**场景B：点击"暂停"按钮**

-   动作：暂停 AI 识别和本地识别队列（本地 MD5 + 元数据不可暂停）

-   反馈：按钮文字变为"继续"，处理中的任务完成当前步骤后暂停

**场景C：点击"取消全部"按钮**

-   动作：取消当前批次所有待处理任务

-   反馈：待处理的任务标记为已取消，已完成的不受影响

**场景D：点击单张图片的"重试"按钮**

-   动作：仅重试该图片失败的处理线

-   反馈：该图片失败的线重置为 pending，其他线不受影响

**5.4 关闭进度面板**

**场景：点击遮罩层 / 按 ESC**

-   触发：\#progressBackdrop 的 onclick 或全局 keydown Escape

-   动作：调用 toggleProgressPanel()，移除 .open 类

-   反馈：面板向下滑出，遮罩淡出

**6. 人物管理**

**6.1 人物卡片**

**组件状态定义**

  ---------- ------------------------------------------------------
  **状态**   **表现**
  Default    圆形头像 + 名称 + 照片数量，无选中效果
  Selected   青色边框 + 右上角出现勾选徽章（青色圆形 + 白色对勾）
  Hover      卡片微上移，头像边框变亮
  ---------- ------------------------------------------------------

**交互逻辑**

**场景A：单击人物卡片（多选）**

-   触发：.person-card 的 onclick 调用 selectPerson(this, id)

-   动作：切换 .selected 类（支持多选）

**场景B：双击人物卡片（按人物筛选）**

-   触发：.person-card 的 ondblclick 调用 filterByPerson(id)

-   动作：调用 switchPage(\'browse\') 跳转到浏览页，按 person\_id 过滤图片网格

**场景C：双击人物名称（内联编辑）**

-   触发：.person-name 的 ondblclick，先 event.stopPropagation() 阻止冒泡到卡片的双击事件

-   动作：设置 contentEditable=true，自动全选文字并聚焦

-   保存：失焦（blur）或按 Enter 时退出编辑模式，将修改后的名称写回 peopleData 数据源

**6.2 合并/拆分操作**

**场景A：合并人物**

-   前置条件：已选中至少 2 个人物卡片

-   动作：点击"合并"按钮，弹出确认对话框，选择保留的名称，系统更新所有关联图片的 person\_id

**场景B：拆分人物**

-   前置条件：已选中 1 个人物卡片

-   动作：点击"拆分"按钮，弹出图片选择对话框，用户勾选需要分离的图片，系统为选中图片创建新 person\_id

**7. 插件管理**

**7.1 插件卡片**

**场景B：切换启用/禁用开关**

-   动作：启用时加载插件模块并注册功能；禁用时卸载模块但保留配置和文件

-   反馈：启用后插件提供的功能立即可用；禁用后相关 UI 入口消失

**8.2 交互逻辑**

**场景A：修改下拉选择项**

-   触发：select 的 onchange 事件

-   动作：更新配置并写入本地存储

-   生效规则："立即生效"的设置立即应用；"仅影响新导入"的设置仅对后续导入的图片生效，已有图片不受影响

**场景B：拖动滑块（最小置信度）**

-   触发：input\[type=range\] 的 oninput 事件

-   动作：实时更新右侧显示的数值（如 0.50）

-   生效：松开鼠标后写入配置

**场景C：切换 Toggle 开关**

-   触发：.toggle 元素的 onclick

-   动作：切换 .on 类，更新配置

**场景D：点击"清理缓存"按钮**

-   动作：立即执行缓存清理（删除超过 30 天未访问的压缩图缓存）

-   反馈：按钮短暂显示 Loading，完成后 Toast 提示"已清理 X MB"

  ------------ ---------------- ------------------------------------------
  **快捷键**   **触发条件**     **动作**
  Escape       始终生效         优先关闭全屏预览 → 导入对话框 → 进度面板
  ← 左箭头     仅预览打开时     切换到上一张图片（循环）
  → 右箭头     仅预览打开时     切换到下一张图片（循环）
  滚轮上/下    仅预览打开时     向上/向下切换图片（循环）
  Enter        人物名称编辑中   确认修改并退出编辑模式
  ------------ ---------------- ------------------------------------------

注意：左右箭头和滚轮仅在全屏预览打开时生效，不会干扰其他页面的正常滚动行为。
